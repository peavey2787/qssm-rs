# MS v2 observable boundary contract

This note freezes the MS v2 reduction target used by the formal crate.

From this point onward, the MS v2 transcript structure and public APIs are treated as fixed. No further feature work should change what is observable from the prover. Only correctness verification, distinguisher analysis, and reduction work should continue.

## Frozen interface

The canonical code surface is:

- `qssm_ms::ValueCommitmentV2`
- `qssm_ms::PredicateOnlyStatementV2`
- `qssm_ms::PredicateOnlyProofV2`
- `qssm_ms::commit_value_v2`
- `qssm_ms::prove_predicate_only_v2`
- `qssm_ms::verify_predicate_only_v2`
- `qssm_ms::simulate_predicate_only_v2`
- `qssm_ms::verify_predicate_only_v2_with_programming`

The formal reduction surface in [truth-engine/qssm-proofs/src/reduction_zk.rs](../../truth-engine/qssm-proofs/src/reduction_zk.rs) treats this accessor set as the authoritative public boundary.

## Observable sigma-algebra

The MS v2 observable sigma-algebra is generated by the following public objects and measurable projections:

### Public statement generators

- public value commitment
- target
- binding entropy
- binding context
- application context

### Public proof projections

- `PredicateOnlyStatementV2::statement_digest()`
- `PredicateOnlyProofV2::result()`
- `PredicateOnlyProofV2::bitness_global_challenges()`
- `PredicateOnlyProofV2::comparison_global_challenge()`
- `PredicateOnlyProofV2::transcript_digest()`

Everything in the formal MS v2 distinguisher suite is evaluated as a measurable function of these generators only.

## Hidden quantities

The following objects are mathematically hidden and must not cross the observable boundary:

- hidden value
- commitment blinders
- prover randomness
- witness object `PredicateWitnessV2`
- internal Sigma announcements / responses that are not exposed by the frozen `qssm_ms` API

If the implementation later exposes any of these quantities through new accessors or a new wire-format guarantee, the zero-knowledge boundary must be redefined before any claim is extended.

## Simulator dependency rule

The witness-free simulator is allowed to depend only on:

- `MsHiddenValuePublicInput`
- fresh simulator coins / simulator seed
- programmable random-oracle responses derived from the public statement

The simulator is not allowed to depend on:

- hidden value
- commitment blinders
- prover randomness
- `PredicateWitnessV2`

In the current codebase this is enforced by construction in [truth-engine/qssm-proofs/src/reduction_zk.rs](../../truth-engine/qssm-proofs/src/reduction_zk.rs): the simulator rebuilds the public statement from the commitment-bearing public input and calls `qssm_ms::simulate_predicate_only_v2` without any witness argument.

## Reduction consequence

The composed QSSM hybrid argument must preserve exactly this observable boundary across all worlds:

- `H0`: real MS v2 prover, real LE prover
- `H1`: simulated MS v2, real LE prover
- `H2`: simulated MS v2, simulated LE prover
- `H3`: fully simulated composed world with programmed oracle queries inlined

This means the MS theorem and the LE theorem are not separate presentation artifacts anymore. They are two transitions inside the same hybrid game over one verifier view.

## Empirical suite scope

The empirical MS-only distinguisher suite in [truth-engine/qssm-proofs/src/reduction_zk.rs](../../truth-engine/qssm-proofs/src/reduction_zk.rs) now checks three layers on this frozen boundary:

- statistical layer: nibble histograms, adjacent-byte correlation / byte-delta checks, entropy estimates, and observable conditional bias
- structure layer: witness-correlated hidden features against public projections
- simulator gap layer: smoothed KL / Jensen-Shannon approximations on stable observable projections

These checks are evidence about the frozen boundary. They are not a substitute for the full ROM indistinguishability proof.
